┌──────────────────────────────────────────────────────────────────────────┐
│ 🧬 WHAT IS RC EQUIVARIANCE?                                              │
└──────────────────────────────────────────────────────────────────────────┘

  DNA is double-stranded:
    Forward:  5'-ATCG-3'
              3'-TAGC-5'
    Rev Comp: 5'-CGAT-3'  (functionally equivalent)

  RC equivariance ensures:
    model(ATCG) ≈ reverse(model(CGAT))

  Benefits:
  ✅ Doubles effective training data
  ✅ Better generalization to unseen strands
  ✅ Strand-agnostic feature learning
  ✅ Critical for genomic tasks

┌──────────────────────────────────────────────────────────────────────────┐
│ 🔧 HOW IT WORKS                                                          │
└──────────────────────────────────────────────────────────────────────────┘

  1. RC Data Augmentation
     - Each batch: 50% of sequences randomly RC-flipped
     - No memory overhead (in-place transformation)
     - Doubles diversity of training data

  2. RC Consistency Loss
     - Computes: f(x) vs. reverse(f(RC(x)))
     - Penalizes inconsistent predictions
     - Weight: 0.1 (tunable)

  3. Combined Loss
     L_total = L_CE + λ·L_RC
     where:
     - L_CE = cross-entropy (standard)
     - L_RC = RC consistency
     - λ = 0.1 (RC weight)

  Edit train_20k_rc_equivariant.py to customize:

  # RC Augmentation Mode
  RC_AUGMENTATION_MODE = 'random'  # Options:
                                   # 'random' - 50% RC flip (default)
                                   # 'double' - add RC copies (2x batch)
                                   # 'none'   - no augmentation

  # RC Loss Weight
  RC_LOSS_WEIGHT = 0.1            # 0.0 to 1.0
                                   # Higher = stronger RC invariance
                                   # Lower = weaker constraint

  # Enable/Disable RC Loss
  USE_RC_LOSS = True              # False = augmentation only
